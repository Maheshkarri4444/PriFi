Deposit Flow
    Off-Chain:

        User signs a fixed message using their wallet
        Wallet derives:
            PrivateKey = H(signature("prifiwallet"))
            zkPublicKey = Poseidon(3, PrivateKey)
            Note: 3 <-- domain seperator
            encPublicKey = EC_Derive(PrivateKey)

        User selects:
            Deposit amount "a"
            Relayer/protocol fee "fee"

        Wallet creates 2 private notes:
            User note:
                (a - fee, r1, zkPublicKey)
                > r1 is a fresh randomness
            Relayer fee note:
                (fee, r2, relayerZkPubKey) 
                > r2 is a fresh randomness
        Wallet computes commitments:
            C1 = H(1,a - fee, r1, zkPublicKey)
            C2 = H(1,fee, r2, relayerZkPubKey)

        Wallet encrypts note data:
            ciphertext1 = Encrypt(encPublicKey, (a - fee, r1))
            ciphertext2 = Encrypt(relayerEncPubKey, (fee, r2))
        
        Wallet generates a zk proof proving:
            (a - fee) + fee = deposited ETH amount

        without revealing:
            note amounts
            randomness
            ownership information


    On-Chain:
        Transaction sent to shielded pool contract
        ETH value a is transferred into the pool

        Contract verifies:
            zk proof validity
            commitment correctness
            deposit amount consistency

        Contract inserts:
            C1
            C2
        into the incremental Merkle tree

        Contract emits encrypted note events:
            encryptedNote1
            encryptedNote2
        User and relayer wallets scan emitted events to recover their notes.
